# Mirad TTS Audio Samples

This directory contains audio samples generated by the Mirad text-to-speech pipeline using three backends: eSpeak NG, Piper TTS, and MBROLA.

## Directory Structure

```
samples/
├── espeak/          # Samples generated with eSpeak NG backend
├── piper/           # Samples generated with Piper TTS backend
├── mbrola/          # Samples generated with MBROLA de6 backend
└── README.md        # This file
```

## Backends

### eSpeak NG

- **Backend**: Traditional formant-based synthesis
- **Quality**: Robotic but fast
- **Voice**: Default eSpeak voice
- **File sizes**: 25-39 KB per sample
- **Use case**: Quick testing, low resource usage

### Piper TTS

- **Backend**: Neural network-based synthesis
- **Quality**: Natural-sounding, human-like
- **Voice**: en_US-lessac-medium (American English, male)
- **File sizes**: 16-38 KB per sample
- **Use case**: High-quality output, natural speech

### MBROLA de6

- **Backend**: Diphone synthesis using German de6 voice
- **Quality**: Deterministic, inspectable, moderate naturalness
- **Voice**: de6 (German male)
- **File sizes**: Varies per phrase
- **Use case**: Deterministic output, phoneme-level control, phonetic research
- **Why de6**: Supports Mirad-critical phones including `h`, `S` /ʃ/, `Z` /ʒ/, `tS` /tʃ/, `j`, `w`, and `r` — phones that many other MBROLA voices (e.g. Italian) lack.

## Usage

### Generate New Samples

#### Using eSpeak backend

```bash
cd /mnt/s/projects/Project_earthlingish/Mirad-phonemes-engine
PATH="$(pwd)/.gsd/bin:$PATH" \
ESPEAK_DATA_PATH="$(pwd)/.gsd/share/espeak-ng-data" \
PYTHONPATH=src \
python3 -m mirad_tts.cli --backend espeak --wav samples/espeak/word.wav "word"
```

#### Using Piper backend

```bash
cd /mnt/s/projects/Project_earthlingish/Mirad-phonemes-engine
PYTHONPATH=src \
python3 -m mirad_tts.cli --backend piper --wav samples/piper/word.wav "word"
```

#### Using MBROLA backend

```bash
cd /mnt/s/projects/Project_earthlingish/Mirad-phonemes-engine
PYTHONPATH=src \
python3 -m mirad_tts.cli --backend mbrola --wav samples/mbrola/word.wav "word"

# Generate .pho file for inspection
PYTHONPATH=src \
python3 -m mirad_tts.cli --backend mbrola --pho samples/mbrola/word.pho "word"

# Debug per-word phone diagnostics
PYTHONPATH=src \
python3 -m mirad_tts.cli --backend mbrola --mbrola-debug "Ha Mirad."
```

#### Using generate_samples.py script

```bash
# All default words with a specific backend
python packages/tts/scripts/generate_samples.py --backend mbrola

# Specific words
python packages/tts/scripts/generate_samples.py --backend mbrola Mirad igay "Ha Mirad."

# MBROLA with debug output
python packages/tts/scripts/generate_samples.py --backend mbrola --mbrola-debug
```

### Play Samples

```bash
# Using aplay (ALSA)
aplay samples/mbrola/mirad.wav

# Using ffplay
ffplay samples/mbrola/mirad.wav

# Using vlc
vlc samples/mbrola/mirad.wav
```

### Compare Backends

Listen to the same word with all three backends:

```bash
# eSpeak version
aplay samples/espeak/mirad.wav

# Piper version
aplay samples/piper/mirad.wav

# MBROLA version
aplay samples/mbrola/mirad.wav
```

## Pronunciation Rules Demonstrated

### Consonant Mappings
- **r → ɾ**: Alveolar flap (not trill), as in "Mirad" → ˈmiɾad
- **j → ʒ**: Voiced palatal fricative, as in "jal" → ʒal
- **x → ʃ**: Post-alveolar fricative, as in "tixe" → ˈtiʃe
- **q → k**: Foreign word mapping, as in "qatar" → ˈkataɾ
- **c → t͡ʃ**: Affricate, as in "cema" → ˈt͡ʃema
- **h → h**: Voiceless glottal fricative (de6 supports this, unlike Italian MBROLA voices)

### Vowel Mappings
- **Post-y-glided**: ay→aɪ, ey→eɪ, iy→iɪ, oy→oɪ, uy→uɪ
- **Post-w-glided**: aw→ɔ, ew→ɛʊ, iw→iʊ, ow→oʊ, uw→uʊ
- **Pre-y-glided**: ya→ja, ye→je, yi→ji, yo→jo, yu→ju
- **Pre-w-glided**: wa→wa, we→we, wi→wi, wo→wo, wu→wu
- **Circum-y-glided**: yay→jaɪ, yey→jeɪ, yiy→jiɪ, yoy→joɪ, yuy→juɪ
- **Pre-w-post-y-glided**: way→waɪ, wey→weɪ, wiy→wiɪ, woy→woɪ, wuy→wuɪ

### Stress Assignment
- Stress falls on the **last non-final syllable**
- Final syllables are never stressed
- Complex vowels (glided) count as single syllable nuclei

### MBROLA-Specific Mapping

The MBROLA de6 backend maps vowels with long-vowel notation (e: i: o: u:) and uses `O` for `aw`:

| Mirad | de6 | | Mirad | de6 |
|-------|-----|-|-------|-----|
| a | a | | ya | j a |
| e | e: | | ay | a j |
| i | i: | | aw | O |
| o | o: | | wa | w a |
| u | u: | | yay | j a j |

## Technical Details

### Audio Format
- **Format**: 16-bit mono WAV
- **Sample rate**: 22050 Hz (eSpeak/Piper), 16000 Hz (MBROLA de6)
- **Bit depth**: 16-bit
- **Channels**: 1 (mono)

### MBROLA .pho File Format

Each line in a `.pho` file specifies:

```
PHONE DURATION_MS [PITCH_POSITION_PERCENT PITCH_HZ ...]
```

Example for "Ha Mirad.":

```
_ 80
h 85
a 140 0 115 50 135 100 110
_ 60
m 75
i: 140 0 115 50 135 100 110
r 65
a 110 0 110 100 105
d 65
_ 220
_ 120
```

### Backend Comparison

| Feature | eSpeak NG | Piper TTS | MBROLA de6 |
|---------|-----------|-----------|------------|
| Type | Formant synthesis | Neural synthesis | Diphone synthesis |
| Quality | Robotic | Natural | Deterministic |
| Determinism | Low | Low | **High** |
| Inspectable | No | No | **Yes (.pho files)** |
| Speed | Very fast | Fast | Very fast |
| Resource usage | Low | Moderate | Low |
| Voice variety | Many | Custom models | 30+ voices |
| File size | Larger | Smaller | Varies |
| Best for | Testing | Production | Phonetic research, deterministic output |

## Installation

### eSpeak NG

Installed locally in `.gsd/`:
- Binary: `.gsd/bin/espeak-ng` (v1.51)
- Data: `.gsd/share/espeak-ng-data/`

### Piper TTS

Installed via pip:
- Package: `piper-tts` (v1.4.2)
- Voice model: `.gsd/piper-voices/en_US-lessac-medium.onnx` (61 MB)

### MBROLA

```bash
sudo apt install mbrola mbrola-de6
```

- Binary: `/usr/bin/mbrola`
- Voice database: `/usr/share/mbrola/de6/de6`

## Notes

- All samples are generated from the grammar-anchored test cases
- IPA transcriptions match the Mirad Grammar document
- All three backends use the same phonology rules
- MBROLA de6 is chosen because it supports Mirad `h`, unlike Italian MBROLA voices
- Piper TTS produces more natural-sounding speech
- eSpeak NG is faster and uses fewer resources
- MBROLA provides deterministic, inspectable output with phoneme-level control
